NGUYÊN NHÂN LỖI VÀ HƯỚNG KHẮC PHỤC

Dựa trên log Tomcat mà bạn cung cấp, tôi đã kiểm tra source code và tìm ra chính xác 2 nguyên nhân khiến dự án bị lỗi sau khi merge code:

1. LỖI 1: SAI CÚ PHÁP TRONG FILE JSP (JASPER EXCEPTION)
- Nguyên nhân: Quá trình merge code đã tạo ra các thẻ bị lặp và đóng mở sai vị trí (unbalanced tags) trong file `booking-process.jsp` (tại dòng 507, phần chứa thẻ `<c:when>` và `<c:otherwise>`).
Khi Tomcat dịch file JSP này sang Java, nó báo lỗi `JasperException` và `ClassNotFoundException` vì không thể biên dịch thành công.
- Đã khắc phục: Tôi đã trực tiếp sửa lỗi cú pháp này trong file `src/main/webapp/WEB-INF/views/receptionist/booking-process.jsp`. Code JSP của bạn đã hoàn toàn hợp lệ trở lại!

2. LỖI 2: KHÔNG KẾT NỐI ĐƯỢC DATABASE (SQL SERVER EXCEPTION)
- Nguyên nhân: Log Tomcat báo lỗi: `com.microsoft.sqlserver.jdbc.SQLServerException: Login failed for user 'sa'`.
Điều này có nghĩa là ứng dụng đang không thể đăng nhập vào SQL Server với tài khoản `sa`.
- Cách hướng sửa:
  Bạn hãy mở file: `src/main/java/com/mycompany/hotelmanagement/config/DBContext.java`
  Tìm đến dòng 29 và 30:
  ```java
  config.setUsername("sa");
  config.setPassword("12345");
  ```
  Vui lòng thay đổi `12345` thành mật khẩu SQL Server TRÊN MÁY BẠN. (Hoặc kiểm tra xem SQL Server của bạn có đang mở port 1433 hay không).

BƯỚC CUỐI CÙNG (RẤT QUAN TRỌNG):
Vì Tomcat đã bị lỗi (fail) khi cố gắng dịch file JSP lỗi lúc trước, nó sẽ nhớ cái lỗi `ClassNotFoundException` đó. Bạn BẮT BUỘC phải làm bước sau để ứng dụng nhận file JSP tôi vừa sửa:
1. Trong NetBeans (hoặc IDE bạn đang dùng), chuột phải vào Project -> Chọn "Clean and Build".
2. Tắt hẳn Server Tomcat (Stop).
3. Chạy lại dự án (Run).

Sau khi cập nhật mật khẩu Database và Clean & Build lại, nút "Cập nhật & Duyệt" của bạn sẽ hoạt động hoàn hảo!
